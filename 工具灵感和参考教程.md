# Sub2API 使用教程

Sub2API 是一个 AI API 网关平台，用于统一管理和分发 AI 模型订阅的 API 配额。支持 Claude、OpenAI、Gemini 等主流模型，提供统一的 OpenAI 兼容接口。

***

## 快速开始（Windows 10/11 自动安装）

### 前置要求

- Windows 10 版本 2004 及以上（内部版本 19041+）
- CPU 支持并开启硬件虚拟化（VT-x / AMD-V）
- 至少 4GB 可用内存

### 一键安装

1. **以管理员身份运行 PowerShell**：右键开始菜单 → Windows PowerShell (管理员)
2. **进入项目目录**：
   ```powershell
   cd e:\Git\sub2api
   ```
3. **运行自动安装脚本**：
   ```powershell
   .\auto-install.ps1
   ```
   脚本会自动完成：
   - 安装 WSL2 和 Ubuntu
   - 安装 Docker Desktop
   - 配置国内 Docker 镜像加速
   - 部署 Sub2API 容器
   - 生成管理员密码
4. **访问管理后台**：
   安装完成后，打开浏览器访问：`http://localhost:8787`

### 自定义参数

```powershell
# 自定义端口
.\auto-install.ps1 -ServerPort 8080

# 指定管理员密码
.\auto-install.ps1 -AdminPassword "your-password"

# 同时安装 Codex CLI
.\auto-install.ps1 -InstallCodex

# 重置安装状态，从头开始
.\auto-install.ps1 -Reset
```

### 断点续传

脚本支持断点续传。如果安装过程中断（如重启电脑），再次运行脚本即可从上次的位置继续。

## 手动部署（Docker Compose）

### 前置要求

- Docker 20.10+
- Docker Compose v2+

### 部署步骤

1. **创建部署目录**：
   ```bash
   mkdir -p sub2api-deploy && cd sub2api-deploy
   ```
2. **下载部署脚本**：
   ```bash
   curl -sSL https://raw.githubusercontent.com/Wei-Shaw/sub2api/main/deploy/docker-deploy.sh | bash
   ```
3. **启动服务**：
   ```bash
   docker compose up -d
   ```
4. **查看日志**（获取管理员密码）：
   ```bash
   docker compose logs sub2api | grep "admin password"
   ```
5. **访问管理后台**：
   打开浏览器访问：`http://localhost:8080`

### 手动配置

如果需要自定义配置：

```bash
# 编辑环境变量
nano .env

# 生成安全密钥
openssl rand -hex 32
```

`.env` 关键配置项：

```bash
# PostgreSQL 密码
POSTGRES_PASSWORD=your_secure_password

# JWT 密钥（重启后保持登录状态）
JWT_SECRET=your_jwt_secret

# 管理员账号（可选，不设置则自动生成）
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=your_password

# 自定义端口
SERVER_PORT=8080
```

## 基本使用流程

### 1. 登录管理后台

访问 `http://localhost:8787`（或你配置的端口），使用管理员账号登录。

### 2. 添加渠道

在左侧菜单选择「渠道管理」→「添加渠道」，填入你的 API Key：

- **OpenAI**：`sk-xxx`
- **Claude**：`sk-ant-xxx`
- **DeepSeek**：`sk-xxx`
- **豆包**：`sk-xxx`
- **通义千问**：`sk-xxx`
- 其他支持 OpenAI 格式的厂商

### 3. 创建 API 密钥

在左侧菜单选择「API 密钥」→「创建密钥」，生成用于调用的密钥。

### 4. 对接应用

在支持 OpenAI 格式的应用中配置：

```
接口地址：http://localhost:8787/v1
API 密钥：你生成的密钥
```

### 5. 监控用量

在「仪表盘」查看实时调用统计、Token 用量和成本。

## 常用命令

### 服务管理

```bash
# 启动服务
docker compose up -d

# 停止服务
docker compose down

# 重启服务
docker compose restart

# 查看状态
docker compose ps

# 查看日志
docker compose logs -f sub2api
```

### 升级版本

```bash
# 拉取最新镜像
docker compose pull

# 重建容器
docker compose up -d
```

### 数据备份

```bash
# 备份整个部署目录
tar czf sub2api-backup-$(date +%Y%m%d).tar.gz sub2api-deploy/

# 恢复备份
tar xzf sub2api-backup-20260714.tar.gz
cd sub2api-deploy
docker compose up -d
```

## 进阶配置

### Nginx 反向代理

如果需要通过 Nginx 反向代理，添加以下配置：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:8787;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 必须开启，否则粘性会话失效
        underscores_in_headers on;
    }
}
```

### 简易模式

适合个人开发者或内部团队，不依赖完整 SaaS 功能：

```bash
# 设置环境变量
export RUN_MODE=simple
```

### 数据迁移

使用 `docker-compose.local.yml` 部署时，迁移非常简单：

```bash
# 源服务器
docker compose down
cd ..
tar czf sub2api-complete.tar.gz sub2api-deploy/

# 传输到新服务器
scp sub2api-complete.tar.gz user@new-server:/path/

# 新服务器
tar xzf sub2api-complete.tar.gz
cd sub2api-deploy/
docker compose up -d
```

## 常见问题

### 1. Docker 启动失败：虚拟化未开启

**原因**：CPU 硬件虚拟化未开启

**解决**：

1. 重启电脑进入 BIOS
2. 开启 Intel Virtualization Technology（Intel CPU）或 SVM Mode（AMD CPU）
3. 保存并重启

### 2. 镜像拉取超时

**原因**：国内网络无法访问 Docker 官方镜像仓库

**解决**：配置国内镜像加速

```json
// Docker Desktop → Settings → Docker Engine
{
  "registry-mirrors": [
    "https://docker.xuanyuan.me",
    "https://docker.1ms.run",
    "https://docker.m.daocloud.io"
  ]
}
```

### 3. 无法访问 8080 端口

**排查步骤**：

1. 确认容器运行状态：`docker compose ps`
2. 查看日志：`docker compose logs sub2api`
3. 检查端口占用：`netstat -ano | findstr :8080`
4. 检查防火墙设置

### 4. WSL2 安装失败

**解决**：

1. 确保 Windows 版本 ≥ 2004
2. 启用 Windows 功能：WSL 和虚拟机平台
3. 更新 WSL2 内核：<https://aka.ms/wsl2kernel>
4. 重启电脑

### 5. 管理员密码忘记了

```bash
# 查看自动生成的密码
docker compose logs sub2api | grep "admin password"

# 或重置密码（修改 .env 后重启）
docker compose down
nano .env  # 修改 ADMIN_PASSWORD
docker compose up -d
```
